Guide to writing content files

Escaping characters

Nsm will escape the following characters: ˜, !, @, #, %, ^, &, *, ?, <, >. Just put \character-to-escape, for example to escape tilde use \˜.

Note: nsm does not, and will not, escape \$. This is because it conflicts with MathJax escaping \$. If you do not want to use MathJax on your site and want \$ escaped then manually uncomment the relevant code in PageBuilder.cpp before compiling and installing nsm.

Inserting date, time and timezone information

The date and time that a page was built may be inserted using @currentdate and @currenttime. The timezone this date and time information is associated with may also be inserted using @timezone.

Inserting text from other files

Text may be inserted from another file using @input(path-to-file), where path-to-file is the path from the main site directory to the input file.

For example suppose inside your website directory you have content/footer.content: 

					Page was last modified on @currentdate.

To insert content.footer.content into another file write: 

					@input(content/footer.content)

Note: nsm will not let you create an input loop, for example you cannot have @input(page.template) inside page.template, or any of the files that page.template inputs, or any of the files that they input, and so on.

Paths from pages to other files

It is possible for pages $A$ and $B$ to be located in distinct directories yet input text from the same content file $X$. Consequently if content file $X$ contains a path to another file $Y$, you have the problem that the path from $A$ to $Y$ is not the same as the path from $B$ to $Y$. You can solve this non-constant path problem by using @pathto(path-to-file) where path-to-file is the path from the main site directory to file $Y$. When building a page, nsm will replace @pathto(path-to-file) with the path from the page being built to file $Y$.

An example of when users may find this useful is when writing menus for page templates, or adding local links for any template files.

Including css files

You can include a css file using @cssinclude(path-to-css-file), where path-to-css-file is the path from the main site directory to the css file (not the path from the page to the css file). When building a page, nsm will replace @cssinclude(path-to-css-file) with <link rel='stylesheet' type='text/css' href='path-from-page-to-css-file'>.

Including js files

You can include a js file using @jsinclude(path-to-js-file), where path-to-js-file is the path from the main site directory to the js file (not the path from the page to the js file). When building a page, nsm will replace @jsinclude(path-to-js-file) with: 

					<script type="text/javascript" 
						src="path-from-page-to-js-file">
					</script>

Including a favicon file

You can include a favicon file using @faviconinclude(path-to-favicon-file), where path-to-favicon-file is the path from the main site directory to the favicon file (not the path from the page to the favicon file). When building a page, nsm will replace @faviconinclude(path-to-favicon-file) with <link rel='icon' type='image/png' href='path-from-page-to-favicon-file'>.
